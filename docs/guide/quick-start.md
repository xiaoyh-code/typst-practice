# 快速入門 / Quick Start

## 什麼是 Typst 文件？ / What is a Typst Document?

<div class="bilingual-block">
<div class="zh-block">
<span class="lang-label">中文</span>

Typst 文件以 `.typ` 為副檔名，使用類似 Markdown 的輕量級標記語法。你只需要撰寫純文字，Typst 就會自動處理排版。

最基本的 Typst 文件可以只有一行文字：

```typst
Hello, Typst！
```

這就是一個完整的、可以編譯成 PDF 的有效 Typst 文件！

</div>
<div class="en-block">
<span class="lang-label">English</span>

Typst documents use the `.typ` file extension and employ a lightweight markup syntax similar to Markdown. You only need to write plain text, and Typst handles the typesetting automatically.

The simplest Typst document can be just one line:

```typst
Hello, Typst!
```

This is a complete, valid Typst document that can be compiled into a PDF!

</div>
</div>

## 基本結構 / Basic Structure

<div class="bilingual-block">
<div class="zh-block">
<span class="lang-label">中文</span>

一個典型的 Typst 文件包含以下元素：

- **標題**：使用 `=` 建立標題，`=` 的數量決定層級
- **段落**：空一行分隔段落
- **列表**：使用 `-` 建立無序列表，`+` 建立有序列表
- **強調**：使用 `_文字_` 斜體，`*文字*` 粗體

```typst
= 第一章
== 第一節

這是第一個段落。
空一行會產生新段落。

- 項目一
- 項目二
- 項目三

這是 _斜體_ 和 *粗體* 的示範。
```

</div>
<div class="en-block">
<span class="lang-label">English</span>

A typical Typst document includes the following elements:

- **Headings**: Use `=` to create headings; the number of `=` determines the level
- **Paragraphs**: Separate paragraphs with a blank line
- **Lists**: Use `-` for unordered lists, `+` for ordered lists
- **Emphasis**: Use `_text_` for italics, `*text*` for bold

```typst
= Chapter 1
== Section 1

This is the first paragraph.
A blank line creates a new paragraph.

- Item one
- Item two
- Item three

This is _italic_ and *bold* text.
```

</div>
</div>

## 使用函數 / Using Functions

<div class="bilingual-block">
<div class="zh-block">
<span class="lang-label">中文</span>

Typst 的核心概念之一是 **函數**。函數使用 `#` 符號呼叫，參數放在括號內：

```typst
#text(size: 18pt, fill: red)[紅色大字]
```

這會產生一段紅色、大小為 18pt 的文字。`text` 是函數名稱，`size` 和 `fill` 是參數，`[紅色大字]` 是內容區塊。

你也可以使用 **set 規則** 來設定預設值：

```typst
#set text(size: 14pt, font: "Noto Sans TC")
```

</div>
<div class="en-block">
<span class="lang-label">English</span>

One of Typst's core concepts is the **function**. Functions are called using the `#` symbol, with parameters inside parentheses:

```typst
#text(size: 18pt, fill: red)[Big Red Text]
```

This produces red text sized at 18pt. `text` is the function name, `size` and `fill` are parameters, and `[Big Red Text]` is a content block.

You can also use **set rules** to configure defaults:

```typst
#set text(size: 14pt, font: "Noto Sans TC")
```

</div>
</div>

## 標記 vs 程式碼模式 / Markup vs Code Mode

<div class="bilingual-block">
<div class="zh-block">
<span class="lang-label">中文</span>

Typst 有兩種模式：

- **標記模式 (Markup)**：撰寫一般文字、標題、列表的地方，直接輸入即可
- **程式碼模式 (Code)**：使用 `#` 開頭，可以撰寫變數、運算、函數呼叫等

在程式碼模式中，你可以在區塊 `[]` 中嵌套標記內容：

```typst
#let name = "Typst"
這是一個由 #name 產生的文件。

#text(fill: blue)[
  這段文字是藍色的，
  而且 _可以_ 使用 *標記*！
]
```

</div>
<div class="en-block">
<span class="lang-label">English</span>

Typst has two modes:

- **Markup mode**: Where you write regular text, headings, lists — just type normally
- **Code mode**: Starts with `#`, used for variables, computations, function calls, etc.

In code mode, you can nest markup content inside blocks `[]`:

```typst
#let name = "Typst"
This is a document generated by #name.

#text(fill: blue)[
  This text is blue,
  and you _can_ use *markup*!
]
```
